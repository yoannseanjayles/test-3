# Phase 1.2 — Architecture des APIs

> **Livrable** : Architecture des APIs (choix, comparatif, justifications)
> **Statut** : ✅ Validé HITL le 2026-07-15 (décision D5)
> **S'appuie sur** : D1 (BFF + cache 3 étages + miroir local), D3 (hébergement vidéo free tier), D4 (vision produit v2, hypothèse H8)
> **Contrainte fondatrice** : exclusivement des APIs **gratuites ou disposant d'un plan gratuit**.

## 2. Comparatif des APIs candidates

### 2.1 Métadonnées catalogue (B1-B4, B6, B7, B9)

| API | Gratuité | Couverture | FR | Images | Trailers | Discover/facettes | Recos | Limites |
|---|---|---|---|---|---|---|---|---|
| **TMDB** ✅ | Clé gratuite (usage non commercial) ; ~40-50 req/s | 1 M+ films, 180 k+ séries, personnes, épisodes | ✅ Excellente (communauté FR active) | ✅ CDN natif, 6 tailles, affiches/backdrops/profils | ✅ IDs YouTube par titre | ✅ `/discover` : 30+ filtres AND/OR | ✅ `/recommendations` (collaboratif) + `/similar` | **Non-commercial only** → risque R1 |
| OMDb | 1 000 req/jour | IMDb-like, films surtout | ⚠️ EN only | ⚠️ 1 affiche | ❌ | ❌ (lookup seul) | ❌ | Données IMDb : zone grise juridique en commercial |
| TVmaze | Sans clé, ~20 req/10 s | TV uniquement | ⚠️ EN | ✅ | ❌ | ⚠️ basique | ❌ | Licence CC BY-SA (attribution + share-alike) |
| Trakt | Clé gratuite | Films + TV (IDs, pas d'images) | ⚠️ | ❌ (renvoie vers TMDB) | ❌ | ⚠️ | ✅ orientées utilisateur | Pensée pour le tracking, pas le catalogue |
| Wikidata/Wikipédia | Illimité (fair use) | Encyclopédique | ✅ | ⚠️ Commons | ❌ | ❌ (SPARQL complexe) | ❌ | Latence/complexité SPARQL |

**Verdict** : **TMDB est sans rival** sur le rapport couverture/richesse/gratuité — il couvre à lui seul B1, B2, B3, B4, B6, B7 et B9. C'est la confirmation formelle de l'hypothèse H1 (1.0). Les autres sont au mieux des **compléments** (§3.4).

### 2.2 Disponibilité streaming (B5)

| Solution | Gratuité | Couverture | Verdict |
|---|---|---|---|
| **TMDB `watch/providers`** ✅ | Inclus dans la clé TMDB ; données fournies par JustWatch, **attribution JustWatch obligatoire** | Par pays (FR ✅), par type (flatrate/rent/buy), logos fournis | ✅ **Choix v1** : zéro coût, zéro clé supplémentaire, même modèle de données |
| Watchmode | 2 500 req/mois (non commercial) | 200+ services, deeplinks épisode | ⚠️ Complément possible (deeplinks) mais quota très faible → v2 |
| JustWatch API officielle | Réservée aux « bigger partners » (contact commercial) | La référence du marché | ❌ Inaccessible en v1 ; piste partenariat v2+ |
| Streaming Availability (Movie of the Night) | Free plan (toutes features) | 66 pays | ⚠️ Alternative de secours si TMDB providers insuffisant |

**Verdict** : TMDB `watch/providers` couvre B5 sans coût ni intégration supplémentaire. Les deeplinks directs (lien profond vers le titre chez le streamer) manquent — assumé en v1 (lien vers la page d'accueil du service), Watchmode en piste v2.

### 2.3 Bandes-annonces (B4)

**YouTube embeds** via les IDs vidéo fournis par TMDB (`/videos`) : gratuit, illimité, légal (lecteur embarqué officiel, `youtube-nocookie.com` + façade au clic pour RGPD et performance — pas de quota YouTube Data API nécessaire puisque TMDB fournit les IDs).

### 2.4 Contenu visionnable (B8 — hypothèse H8 validée en D4)

| Source | Gratuité | Contenu | Intégration |
|---|---|---|---|
| **Internet Archive** ✅ | API gratuite, sans clé | Milliers de films du domaine public (classiques, film noir, serials), collections `feature_films` | Advanced Search API (JSON) + fichiers MP4/torrent directs ; métadonnées croisables avec TMDB via titre/année |
| **Open movies (Blender Foundation, etc.)** ✅ | Licences CC | ~15 courts/moyens métrages de démonstration en haute qualité | Fichiers sources directs → notre pipeline HLS (D3) |
| Wikimedia Commons | Gratuit | Vidéos CC/PD éparses | Ponctuel, curation manuelle |

**Verdict** : Internet Archive = colonne vertébrale du catalogue visionnable v1 ; les open movies servent de vitrine qualité (1080p propre) pour démontrer le lecteur premium. La **curation** (sélection des titres IA de qualité suffisante, croisement avec les fiches TMDB) est une tâche éditoriale du back-office (Phase 7).

## 3. Architecture retenue

### 3.1 Vue d'ensemble

```
Client (front)
   │  (jamais de clé API tierce — principe D1)
   ▼
BFF / API interne
   │── Cache Redis (TTL jitterés : config 7 j · fiches 24 h · trending 1 h · providers 24 h)
   │── Miroir PostgreSQL (entités consultées, alimenté incrémentalement :
   │      TMDB Daily ID Exports + /changes ; index Internet Archive curaté)
   ▼
APIs tierces
   ├── TMDB ............ catalogue, images, discover, recos, trending, watch/providers, IDs trailers
   ├── Internet Archive . recherche + fichiers du catalogue domaine public
   └── (v2 : Watchmode deeplinks · TVmaze calendriers · OMDb notes)
Médias
   ├── Images : proxy/transformation depuis image.tmdb.org (AVIF/WebP, srcset)
   ├── Trailers : YouTube embed (nocookie + façade)
   └── Vidéo propre : pipeline HLS sur hébergement free tier (décision 1.3)
```

### 3.2 Gestion des quotas et résilience (rappel D1, appliqué aux choix)

- **Request coalescing** + rate limiter propre côté BFF (budget global < 40 req/s TMDB, alerte à 50 %) ;
- `stale-while-revalidate` + `stale-if-error` : le site survit à une panne TMDB/IA ;
- Objectif O7 : < 1 requête tierce pour 10 requêtes utilisateur (le miroir local + cache y pourvoient) ;
- Monitoring des quotas exposé dans le back-office (Phase 7).

### 3.3 Attributions obligatoires (conformité)

- **TMDB** : logo + mention « This product uses the TMDB API but is not endorsed or certified by TMDB » (footer + page À propos) ;
- **JustWatch** : attribution sur chaque module « Où regarder » (condition d'accès aux données providers) ;
- **Internet Archive / licences CC** : mention de la source et de la licence sur chaque fiche de titre visionnable.

### 3.4 Compléments explicitement reportés (v2)

TVmaze (calendriers de diffusion TV), OMDb (notes IMDb/RT — sous réserve d'analyse juridique), Watchmode (deeplinks), Trakt (import d'historique utilisateur). Aucun n'est nécessaire pour le périmètre Must de la v1 — les intégrer maintenant ajouterait des clés, des quotas et des points de panne sans valeur immédiate.

## 4. Coûts futurs et risques ⚠️

| Risque | Détail | Mitigation |
|---|---|---|
| **R1 — Licence commerciale TMDB** (le plus important) | La clé gratuite TMDB est **non commerciale**. Dès que la publicité génère un revenu effectif, une licence commerciale est requise (~**149 $/mois** pour < 1 M$ de CA annuel, tarif 2026). | Développement et lancement pré-revenus sous clé gratuite ; la bascule payante est **déclenchée par l'activation des revenus AVOD** et couverte par eux (à intégrer au Business Model Canvas, Phase 8). L'architecture n'en dépend pas. |
| R2 — Changement des conditions TMDB | Hausse de tarif, restriction d'endpoints | Miroir local incrémental (D1) = la donnée consultée nous reste accessible ; couche d'abstraction « provider » dans le BFF pour brancher une source alternative |
| R3 — Retrait des données providers (JustWatch via TMDB) | Watch providers est une donnée tierce dans TMDB | Fallback : Streaming Availability API (free plan) ou Watchmode |
| R4 — Qualité inégale d'Internet Archive | Encodages anciens, métadonnées pauvres | Curation éditoriale (whitelist) + ré-encodage HLS via notre pipeline (1.3) |
| R5 — Quotas free tier vidéo | Cf. H9 (addendum) | Monitoring Phase 7, bascule self-host préparée (D3) |

## 5. Synthèse de la décision proposée

1. **TMDB** = source unique principale (catalogue, métadonnées FR, images, discover, recos, trending, trailers-IDs, watch providers) — un seul contrat de données, une seule clé, richesse inégalée à coût nul ;
2. **YouTube embed** pour les bandes-annonces (IDs TMDB, façade RGPD/perf) ;
3. **Internet Archive + open movies** pour le catalogue visionnable v1 (H8), avec curation éditoriale ;
4. **Aucune autre API en v1** — TVmaze/OMDb/Watchmode/Trakt documentés comme extensions v2 ;
5. **Risque R1 accepté et tracé** : gratuité totale en développement/pré-revenus ; licence commerciale TMDB (~149 $/mois) budgétée dans le modèle économique dès activation des revenus (Phase 8).

## Hypothèses signalées

- **H11** : le lancement se fait en pré-revenus (clé TMDB gratuite légitime) ; l'activation AVOD déclenche la licence commerciale TMDB — séquencement à confirmer en Phase 8.
- **H12** : la qualité du sous-ensemble curaté d'Internet Archive est suffisante pour une expérience « premium » après ré-encodage HLS ; à valider par un test réel en 1.3 (proof of concept pipeline).
