## 3. Cartographie des emplacements publicitaires (D6 — tous désactivables)

| Emplacement | Pages | Format | Flag |
|---|---|---|---|
| `ad.preroll` | Watch | Vidéo VAST ≤ 30 s skippable 5 s | `ads.video.preroll` |
| `ad.display.home` | Accueil (entre rails, jamais au-dessus du hero) | Display responsive | `ads.display.home` |
| `ad.display.browse` | Films/Séries/Genres (1 par ~4 rangées de grille) | Display in-grid | `ads.display.browse` |
| `ad.native.rail` | Accueil/Découvrir (rail « Sponsorisé » étiqueté) | Carte native | `ads.native.rail` |
| `ad.display.fiche` | Fiches (sous le casting, jamais entre CTA et synopsis) | Display | `ads.display.fiche` |
| — interdits — | Résultats de recherche organiques, espace utilisateur, pages légales, admin | (neutralité D4 + confiance) | — |

`ads.enabled` (global) **ET** flag d'emplacement **ET** consentement TCF requis pour l'affichage. Pub OFF = aucun module chargé (D6/D7).

## 4. Règles SEO transverses (rappel D1, appliquées au sitemap)

- **Indexables** : accueil, hubs, grilles, genre×type, fiches, saisons, personnes, listes, `/gratuit`, watch, support ; **noindex** : recherche, tris, combinaisons de filtres multiples (canonical vers la vue de base), espace utilisateur, admin ;
- **Sitemaps XML segmentés** : `sitemap-films.xml`, `sitemap-series.xml`, `sitemap-personnes.xml`, `sitemap-pages.xml` (< 50 k URLs chacun, `lastmod` fiables), générés depuis le miroir local ;
- **Fil d'Ariane** sur fiches/genres (`BreadcrumbList`) : `Accueil > Films > Science-fiction > Dune` ;
- Pagination : « Load More » + URLs `?page=n` crawlables auto-canoniques.

## Hypothèses signalées

- **H15 (rattachée à D8)** : l'ingestion vidéo est **réservée aux administrateurs** en v1 — pas d'UGC. Si vous souhaitez l'upload par les utilisateurs, c'est un changement de périmètre majeur (modération, cadre légal, stockage) à décider explicitement.
- **H16** : libellé « Gratuit ▶ » pour l'entrée du catalogue visionnable — naming à affiner avec l'identité (Phase 4).
- **H17** : URLs en français (`/regarder`, `/ma-liste`) cohérentes avec le marché FR (H6) ; la structure i18n (préfixe `/en/…`) reste prête.
