# Phase 1.5 — Parcours Utilisateur (User Flows)

> **Livrable** : User Flows (visiteur, utilisateur connecté, administrateur)
> **Statut** : ✅ Validé HITL le 2026-07-15 (décision D12)
> **S'appuie sur** : D1 (UX), D4 (vision), D6 (pub ON/OFF + pré-roll), D7 (streaming), D8 (ingestion), D10 (sitemap), D11 (UGC avec interrupteur)
> Convention : `[pub ON]` = étape présente uniquement si publicité activée (D6) ; `[UGC ON]` = uniquement si `ugc.upload.enabled` (D11).

---

## 1. Visiteur (non connecté)

### V1 — Découverte (arrivée directe)
```
Accueil → hero + rails (Tendances, Gratuit ▶, éditoriaux)
  → scroll/rail → carte titre (hover preview différé) → Fiche
  → décision : Bande-annonce (modale) · Où regarder (sortie trackée) · ▶ Regarder (si visionnable)
```
- Aucun mur de connexion sur la découverte ; les actions personnelles (favoris) déclenchent une **invite de connexion contextuelle** (« Créez un compte pour retrouver votre liste partout »), jamais bloquante en amont.
- Bannière de consentement **TCF** à la première visite (refuser = navigation intacte, pub contextuelle seulement `[pub ON]`).

### V2 — Recherche
```
Icône / raccourci "/" → saisie → suggestions instantanées (6-8, affiche+année+type)
  → Entrée = page Résultats (onglets Films/Séries/Personnes, facettes)
  → zéro résultat impossible : correction typo + tendances proposées (D1)
```

### V3 — Arrivée SEO (le flux d'acquisition principal, O2)
```
Google (« où regarder X », « casting de Y », « regarder Z gratuitement »)
  → Fiche ou page Watch (réponse above-the-fold immédiate)
  → maillage : similaires, genre, listes → poursuite de session (V1)
```

### V4 — Lancement visionnage (catalogue gratuit)
```
Fiche ou /gratuit → ▶ Regarder → page Watch
  → [pub ON + consentement] pré-roll VAST ≤ 30 s, skippable à 5 s (D6)
  → lecture HLS adaptative, sous-titres, contrôles complets (D7)
  → progression sauvegardée en localStorage (visiteur)
  → à la fermeture/fin : invite douce « Créez un compte pour reprendre où vous en êtes »
  → écran de fin : recos post-visionnage annulables
```

### V5 — Conversion (inscription)
Déclencheurs contextuels : ajout favori (V1), sauvegarde de progression (V4), filtre « Mes services ».
```
/inscription (e-mail+mdp) → onboarding /bienvenue : 3 genres + 3 titres + mes services (skippable)
  → retour à la page d'origine avec l'action initiale accomplie (le favori est ajouté, la progression migrée du localStorage)
```

## 2. Utilisateur connecté

### U1 — Session type « ce soir » (persona zappeur, O1 < 2 min)
```
Accueil → rail 1 « Continuer la lecture » (reprise 1 clic)
  ou rail « Parce que vous avez ajouté X » (explicable, D1)
  → Fiche → ▶ Regarder / Où regarder → décision < 2 min
```

### U2 — Favoris / Watchlist
```
Toute carte/fiche → ♥ (1 clic, optimistic UI) → Ma liste/Favoris
  (tri, filtres, retrait ; signal central des recos ; badge « désormais gratuit » si le titre devient visionnable)
```

### U3 — Reprise cross-device
```
Lecture mobile (heartbeat 15 s) → arrêt → ouverture desktop
  → Accueil : « Continuer la lecture » avec barre de progression → reprise à ±15 s
  Seuils : <5 % → du début · >92 % → marqué vu, retiré du rail (D7)
```

### U4 — Historique & RGPD
```
/ma-liste/historique → consultés / vus / en cours
  → purge unitaire ou totale · /parametres → export des données · suppression de compte
  (double confirmation ; effacement effectif des progressions, favoris, historique)
```

### U5 — Upload d'une vidéo `[UGC ON]` (D11)
```
/studio → « Téléverser » → fichier vidéo
  → déclaration OBLIGATOIRE : je détiens les droits / licence (CC, domaine public, œuvre propre)
  → métadonnées (titre, synopsis, genre, langue, sous-titres .srt/.vtt facultatifs)
  → envoi (upload résumable) → statut « Encodage » (pipeline D7) → statut « En modération »
  → décision admin (A3) : Publiée (fiche /video/…, visible dans /gratuit) ou Refusée (motif notifié)
  → /studio : stats de vues de mes vidéos ; retrait possible à tout moment
Garde-fous : quota par utilisateur (nombre + Go), formats acceptés listés, aucune publication sans approbation.
```

### U6 — Paramètres & consentements
```
/parametres → langue, taille/contraste sous-titres (appliqué au lecteur)
  → consentements pub/analytics modifiables à tout moment (réouverture CMP)
```

## 3. Administrateur

### A1 — Accès
```
/admin (compte à rôle admin, 2FA) → Dashboard : KPIs, quotas TMDB/R2, jobs d'encodage, alertes, file de modération (compteur)
```

### A2 — Ingestion de contenu (D8)
```
/admin/contenus/ingestion → source : upload fichier OU import Internet Archive (recherche intégrée)
  → vérification licence/source (champ obligatoire, bloquant)
  → encodage HLS (suivi de job) → rattachement fiche TMDB ou fiche manuelle → sous-titres → Publier
  → le titre apparaît dans /gratuit + badge ▶ sur sa fiche
```

### A3 — Modération UGC `[UGC ON]` (D11)
```
/admin/contenus/moderation → file d'attente (aperçu lecteur, métadonnées, déclaration de droits)
  → Approuver (publication) / Refuser (motif obligatoire, notifié à l'utilisateur)
  → signalements post-publication : file dédiée → retrait (notice & takedown, traçé)
  → interrupteur ugc.upload.enabled + quotas par utilisateur
```

### A4 — Publicité (D6)
```
/admin/publicite → kill switch global + flags par emplacement (dont ad.preroll)
  → campagnes : créa (vidéo VAST / display / native), ciblage contextuel (genres), fréquence, période
  → /admin/publicite/revenus : CPM, CTR, complétion (quartiles), revenu/utilisateur
  Garde-fou : tableau « impact UX » (rétention, complétion contenu) à côté de chaque activation (principe 15)
```

### A5 — Utilisateurs & gestion courante
```
/admin/utilisateurs → recherche, rôles, suspension, demandes RGPD (export/suppression)
/admin/contenus → curation éditoriale (listes, mises en avant du hero)
/admin/analytics → audience, visionnage, rétention (Phase 9)
/admin/systeme → quotas API, cache, stockage R2, santé des jobs
```

## 4. Matrice parcours × exigences

| Exigence validée | Couverte par |
|---|---|
| Pub activable/désactivable (D6) | V4, A4 (`[pub ON]` conditionne chaque étape pub) |
| Pré-roll avant chaque vidéo (D6) | V4 (aussi pour utilisateurs connectés, même flux) |
| Héberger/diffuser depuis l'appli (D8) | A2 |
| Upload utilisateur + interrupteur (D11) | U5, A3 (`[UGC ON]`) |
| Reprise cross-device (D7) | U3, migration localStorage → compte en V5 |
| RGPD (brief) | V1 (TCF), U4, U6, A5 |
| Décision < 2 min (O1) | U1, V1-V4 (chaque flux se termine par une action de valeur) |

## Hypothèses signalées

- **H18** : quotas UGC initiaux proposés — 5 vidéos / 10 Go par utilisateur, extensibles par admin ; à ajuster en Phase 7.
- **H19** : la modération est **exclusivement humaine** en v1 (volumes faibles attendus, UGC OFF au lancement) ; un pré-filtrage automatique est une évolution v2.
- **H20** : les vidéos UGC portent le même pré-roll que le reste du catalogue `[pub ON]` ; pas de partage de revenus avec les créateurs en v1 (sujet Phase 8 si l'UGC décolle).
