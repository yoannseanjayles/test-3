## A. Architecture de streaming vidéo

### A.1 Standard de diffusion : streaming adaptatif (ABR)

- **HLS est le protocole par défaut** en 2026 : obligatoire sur iOS/tvOS, supporté partout (navigateurs via MSE, smart TV). DASH reste pertinent surtout pour le DRM Widevine sur certaines cibles — inutile en v1 sans contenu sous DRM.
- Paramètres éprouvés : segments ~6 s, keyframes ~2 s, **H.264 + AAC-LC** pour la compatibilité maximale ; échelle de qualités (« bitrate ladder ») typique 360p → 720p → 1080p, le lecteur bascule automatiquement selon la bande passante mesurée.
- Le low-latency (LL-HLS, chunks CMAF 200-400 ms) ne concerne que le direct — **hors périmètre** d'une plateforme VOD.

### A.2 Hébergement vidéo à coût nul ou quasi nul (contrainte fondatrice)

| Option | Free tier / coût | Verdict |
|---|---|---|
| **YouTube embeds** | Gratuit, illimité | ✅ Standard du marché pour les **bandes-annonces** (c'est ce que TMDB référence). Contrôle limité, pub YouTube possible. |
| **Cloudflare Stream** | Free tier limité (déjà avantageux si plan Cloudflare existant) ; ensuite ~5 $/1 000 min stockées + 1 $/1 000 min délivrées, encodage inclus | ✅ Meilleur rapport simplicité/coût pour héberger **notre propre contenu** en HLS avec lecteur et miniatures. |
| **Mux** | Free tier développeur (~100 k min délivrées/mois), analytics par seconde | ✅ Alternative « developer-first » crédible ; à comparer formellement en 1.3. |
| **Bunny Stream** | Essai 14 j puis minimum ~1 $/mois | ⚠️ Pas vraiment gratuit, mais le moins cher à volume élevé. Option de croissance. |
| **Self-host (ffmpeg → HLS statique + CDN gratuit)** | 0 € logiciel (open source), coût VPS/stockage seul | ✅ Contrôle total, 0 € de licence ; charge d'exploitation plus élevée (encodage, stockage, purge CDN). Option de repli/évolution. |

**Règle de décision proposée** : YouTube pour les trailers · Cloudflare Stream/Mux (free tier) pour le contenu propre en v1 · bascule self-host ou Bunny si les volumes dépassent les free tiers. Le choix formel est l'objet de l'étape **1.3**.

### A.3 Contenu légalement diffusable avec un budget API gratuit ⚠️ (point stratégique)

Aucune API gratuite ne donne le **droit de diffuser** des films/séries récents sous copyright. Ce que le marché « coût zéro » permet légalement :

1. **Bandes-annonces** : YouTube via les IDs fournis par TMDB — gratuit et légal (embed) ;
2. **Domaine public** : Internet Archive (API gratuite, milliers de films/classiques, collections feature films) ;
3. **Licences ouvertes** : open movies (Blender Foundation…), contenus Creative Commons ;
4. **Orientation légale** pour tout le reste : module « Où regarder » (modèle JustWatch, déjà validé en 1.0).

➡ Le périmètre exact du « visionnable en v1 » est une **décision de vision produit** (traitée dans le 1.1 révisé, hypothèse H8). L'architecture (1.3) doit en revanche être conçue pour accueillir plus tard du contenu licencié (DRM, géo-restrictions) sans refonte.

### A.4 Lecteur vidéo web (open source)

| Lecteur | Poids | Forces | Quand le choisir |
|---|---|---|---|
| **hls.js** | ~80 KB | Standard de facto HLS (multi-millions de dl/semaine), léger | Moteur de lecture HLS pur |
| **Shaka Player** | ~300 KB | DASH + HLS + DRM + offline + Chromecast (Google, utilisé par YouTube TV) | Si DRM/DASH requis (v2+) |
| **Video.js** | moyen | Écosystème de plugins le plus riche (dont **IMA/publicité**), 700 k+ sites | Si l'insertion publicitaire côté player est prioritaire |
| **Vidstack** | léger | Moderne, composants React prêts, **accessible**, thémable | UI premium sur stack React moderne |

**Pattern recommandé 2026** : un moteur (hls.js) + une couche UI moderne et accessible (Vidstack ou UI maison), plutôt qu'un monolithe. Choix formel en 1.3.

### A.5 Fonctions de visionnage premium (standards du secteur)

- **Reprise de lecture** : heartbeat de position toutes les 10-30 s vers le backend (throttlé + `sendBeacon` à la fermeture), reprise cross-device ; seuils usuels : < 2-5 % = recommencer, > 90-95 % = considéré vu.
- **« Continuer la lecture »** : rail n°1 de l'accueil pour les utilisateurs engagés (ordre canonique validé en 1.0).
- **Sous-titres** : format **WebVTT**, multi-pistes, personnalisation (taille/contraste) = exigence d'accessibilité WCAG (et légale dans plusieurs marchés).
- **Contrôles attendus** : lecture/pause, ±10 s, vitesse, qualité (auto + manuel), volume, PiP, plein écran, raccourcis clavier (space, ←/→, F, M), « skip intro » si métadonnées disponibles.
- **Recommandations post-visionnage** : écran de fin avec compte à rebours vers l'épisode suivant / titres similaires — annulable, jamais autoritaire.
- **Sécurité de diffusion** : URLs signées à expiration (soutenu par Cloudflare Stream/Mux nativement), protection hotlink ; le DRM complet (Widevine/FairPlay) n'a de sens qu'avec du contenu licencié (v2+).

## B. Publicité & monétisation (AVOD)

### B.1 Modèles du marché

- **AVOD** (à la demande + pub) : charge publicitaire légère, **4-8 min/heure**, pauses courtes placées aux transitions naturelles — c'est notre modèle de référence.
- **FAST** (chaînes linéaires gratuites) : réplique la TV linéaire, charge pub plus élevée — segment en forte croissance mais hors périmètre v1.
- **Freemium/hybride** : gratuit avec pub + palier premium sans pub — trajectoire cible (Phase 8).

### B.2 Règles UX publicitaires (la pub ne dégrade jamais l'expérience)

1. **Pré-roll** : format le plus efficace (attention maximale, meilleur taux de complétion) ; 1 seul, court (≤ 15-30 s), skippable après 5 s (standard du marché) ;
2. **Mid-roll** : uniquement aux ruptures naturelles de scène et seulement sur les contenus longs ; fréquence proportionnelle à la durée ; jamais sur un contenu < ~15 min en v1 ;
3. **Frequency capping** systématique (éviter la fatigue publicitaire) ;
4. **Display/native** : cartes sponsorisées **clairement étiquetées « Sponsorisé »**, même gabarit que les cartes contenu, jamais dans les résultats de recherche organiques (neutralité validée en 1.0), jamais de pop-up/interstitiel bloquant, jamais d'autoplay sonore ;
5. **Comptage honnête** : une impression = format visible ; complétion vidéo mesurée aux quartiles (0/25/50/75/100 %).

### B.3 Standards techniques

- **VAST** (description d'une pub vidéo) et **VMAP** (planification des pauses) : standards IAB incontournables — permettent de brancher n'importe quelle régie ;
- **Google IMA SDK** (gratuit) : intégration de référence côté player (plugins Video.js/hls.js existants) ; alternatives open source à comparer en 6.2 ;
- **RGPD** : bannière de consentement conforme **TCF v2.2** (CMP), pas de publicité personnalisée sans consentement explicite, pub contextuelle (par genre/contenu) en solution de repli sans cookie — parfaitement adaptée à une plateforme cinéma.

### B.4 Métriques de référence

| Métrique | Définition | Usage |
|---|---|---|
| **CPM** | Revenu pour 1 000 impressions | Pilotage régie/campagnes |
| **CTR** | Clics/impressions | Qualité du ciblage et des créas |
| **Taux de complétion vidéo (VCR)** | % de pubs vues à 100 % | Qualité d'insertion (quartiles VAST) |
| **ARPU / revenu par utilisateur** | Revenu total / utilisateurs actifs | Équilibre charge pub ↔ rétention |
| **Ad load** | Minutes de pub / heure de contenu | Garde-fou UX (≤ 4-8 min/h) |

## C. Principes directeurs additionnels (11-15)

En complément des 10 principes validés en D1 :

11. **HLS + lecteur open source** (moteur hls.js + UI accessible) ; architecture player prête pour DRM sans refonte.
12. **Hébergement vidéo sur free tier** (YouTube pour trailers, Cloudflare Stream/Mux pour contenu propre), URLs signées, bascule self-host possible.
13. **Visionnage premium complet** : reprise cross-device, « Continuer la lecture », WebVTT multi-langues, raccourcis clavier, recommandations post-visionnage annulables.
14. **Publicité plafonnée et honnête** : ≤ 4-8 min/h, pré-roll skippable, mid-roll aux ruptures naturelles uniquement, native ads étiquetées, jamais dans les résultats organiques, VAST/VMAP + consentement TCF.
15. **La monétisation ne précède jamais l'expérience** : tout placement publicitaire passe un critère de non-dégradation UX avant d'être activé (métriques garde-fous : rétention, complétion contenu).

## Hypothèses signalées

- **H8** *(stratégique — tranchée dans le 1.1 révisé)* : le catalogue **visionnable** en v1 = bandes-annonces + domaine public/licences ouvertes ; le reste du catalogue est en découverte + orientation « Où regarder ». Aucune API gratuite ne confère de droits de diffusion sur du contenu récent sous copyright.
- **H9** : volumes v1 compatibles avec les free tiers vidéo (Cloudflare Stream / Mux ~100 k min délivrées/mois) ; à surveiller via monitoring des quotas (Phase 7).
- **H10** : monétisation v1 = AVOD léger (display/native + pré-roll) ; le premium payant (suppression pub) est une évolution Phase 8, pas un prérequis.
