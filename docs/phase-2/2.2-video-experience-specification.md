# Phase 2.2 — Video Experience Specification

> **Livrable** : Spécification consolidée de l'expérience de visionnage
> **Statut** : ⏳ En attente de validation HITL
> **Consolide** : D7 (architecture lecteur), D17 (page Watch), D16 (épisodes), D6 (pré-roll) — et complète : gestes mobiles, grand écran/TV, file publicitaire, matrice d'états du lecteur.
> Ce document est le **contrat de référence** du lecteur pour les Phases 4 (design), 5 (front) et 6.2 (pub).

---

## 1. Le lecteur — matrice d'états

```
idle → (intent : hover/focus ▶) préchargé → shell affiché (poster)
  → [adBreak?] AD_PLAYING (skippable@5s | error/timeout 3s → skip auto)
  → LOADING (manifest+premier segment) → PLAYING ⇄ PAUSED
  → BUFFERING (>250 ms : spinner discret + downswitch ABR)
  → SEEKING (miniatures de scrubbing)
  → ENDED (écran de fin D17 §5)
  → ERROR (retry ×3 backoff → message + position sauvegardée)
```
Chaque état a une représentation visuelle définie (Phase 4) et des événements analytics (D17 §10).

## 2. Contrôles — desktop (rappel D17 §4, gelé)
Barre : lecture/pause · -10/+10 s · volume mémorisé · sous-titres (pistes + personnalisation inline) · qualité (Auto + manuel) · vitesse 0,5-2× · PiP · plein écran · miniatures scrubbing · raccourcis (`Espace/K ←/→ ↑/↓ F M C ?`) ; contrôles masqués après 3 s de lecture, jamais en pause.

## 3. Mobile & tactile
- **Gestes** : tap = afficher contrôles · double-tap gauche/droite = ±10 s (ondulation visuelle) · balayage vertical gauche = luminosité, droite = volume (H49) · pincer = zoom plein cadre (letterbox ↔ crop, opt-in) ;
- Verrouillage de l'orientation en plein écran (paysage) ; bouton 🔒 anti-touche accidentelle ;
- Cibles ≥ 44 px sur mobile (au-delà du minimum WCAG 24 px) ;
- **Data-aware** : sur réseau cellulaire, plafond qualité par défaut 720p (modifiable) + « épisode suivant » ne s'enchaîne pas en cellulaire sans opt-in ;
- PiP automatique proposé quand l'app passe en arrière-plan (si supporté) ;
- La barre système reste masquée en plein écran ; `safe-area` respectée (encoches).

## 4. Grand écran / TV (H7 : via web responsive)
- **Mode 10-feet léger** : à partir de 1920 px + user-agent TV ou navigation clavier exclusive → contrôles agrandis (+50 %), focus ring épais, navigation 100 % flèches/Entrée/Retour ;
- Compatible cast natif du navigateur (AirPlay/Google Cast via API standard quand présente) — pas d'app TV dédiée en v1 (D4) ;
- L'écran de fin en mode TV : focus par défaut sur « Épisode suivant ».

## 5. Sous-titres & audio (contrat d'accessibilité — EAA)
- WebVTT multi-pistes (FR, EN, VO — D7 §5) ; changement à chaud sans interruption ;
- Personnalisation : taille S/M/L/XL · fond · bord de texte ; aperçu live ; persistée au profil (et localStorage en anonyme) ;
- Position auto-remontée quand les contrôles sont visibles (jamais de chevauchement) ;
- Piste audio par défaut : préférence utilisateur > langue du site > VO ;
- Vitesse modifiée = pitch préservé.

## 6. Reprise & continuité (consolidation D7/D16/D17)
- Heartbeat 15 s + `sendBeacon` ; seuils < 5 % / > 92 % ; reprise cross-device à ±15 s ;
- Toast de reprise non bloquant (« Reprise à 1 h 12 — Reprendre du début ») ;
- **Priorité des positions** : serveur (connecté) > localStorage (anonyme, migré à l'inscription V5) ;
- Épisode suivant : compte à rebours 10 s annulable, OFF possible (Paramètres), jamais en cellulaire sans opt-in (§3) ;
- Mémorisés entre sessions : volume, piste sous-titres/audio, réglages sous-titres, vitesse (par contenu), qualité manuelle (par session).

## 7. Expérience publicitaire in-player (contrat D6 — consolidé)
| Règle | Valeur (défauts pilotables en 6.2/7.1) |
|---|---|
| Formats v1 | **Pré-roll uniquement** (mid/post-roll : v1.5+, D4) |
| Durée | ≤ 30 s, skippable après 5 s |
| Capping | 1/titre/session · pas sur enchaînement épisodes < 30 min · pas sur reprise du jour (D17 §3) |
| Échec/timeout | 3 s → lecture directe, silencieuse |
| Conditions | `ads.enabled` ∧ `ads.video.preroll` ∧ consentement TCF ∧ capping OK |
| UI | compte à rebours honnête, « Publicité » affiché, volume aligné sur le contenu (normalisation -23 LUFS, H50), contrôles réduits (pause + volume + skip) |
| Interdits | pub non skippable > 15 s · pub avant le pré-roll · overlay pendant le contenu · son de pub > contenu |
| Tracking | VAST quartiles + skip + click (D17 §10) |

## 8. Qualité de service (budgets consolidés, gelés)
TTFF < 2 s · rebuffer < 1 % · échec démarrage < 1,5 % · bascule de qualité sans coupure (ABR) · INP page < 200 ms · démarrage 720p puis montée (D17 §7).

## 9. Hors périmètre v1 (explicite)
Téléchargement offline · audio descriptions (v2, souhaitable EAA) · chapitres (v1.5 si métadonnées) · multi-audio par titre au-delà de VO+FR · apps TV natives · DRM (trajectoire v2 — D7).

## Hypothèses signalées
- **H49** : gestes luminosité/volume par balayage — standard des apps vidéo mobiles ; à confirmer en test utilisateur Phase 5 (risque de conflit avec le scroll).
- **H50** : normalisation du volume des pubs à la sonie du contenu (référence -23 LUFS) — exigence qualité intégrée au pipeline pub (6.2).
- **H51** : le mode 10-feet est déclenché par heuristique (viewport + input) — pas de build TV séparé en v1.
