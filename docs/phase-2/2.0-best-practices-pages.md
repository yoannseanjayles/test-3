# Phase 2.0 — Best Practices Summary : Définition des Pages

> **Livrable** : Synthèse des meilleures pratiques par famille de pages
> **Statut** : ✅ Validé HITL le 2026-07-15 (décision D13)
> **Périmètre** : pages catalogue · fiches contenus · lecteur vidéo · pages commerciales/support — avec pour chacune : composants indispensables, éléments premium, erreurs fréquentes, opportunités.
> **Note** : les fondamentaux catalogue/fiches/recherche déjà validés en [1.0](../phase-1/1.0-best-practices-summary.md) (D1) ne sont pas répétés ; cette synthèse les complète au niveau **page**.

---

## 1. Pages catalogue (Accueil, Films, Séries, Genres, Tendances, Nouveautés, Gratuit ▶)

### Composants indispensables
- Hero unique (1 titre, jamais d'auto-défilement — D1) avec CTA double : ▶ Regarder / + Ma liste ;
- Rails ordonnés canoniquement : Continuer la lecture → Tendances/Nouveautés → personnalisés explicables → éditoriaux ;
- Grilles facettées avec compteurs, chips, état URL, « Load More » (D1) ;
- **Skeleton screens** systématiques au chargement.

### Éléments premium
- **Varier les conteneurs** : un hero, puis une étagère (shelf), puis une grille mise en avant — la monotonie « 12 rails identiques » fatigue et affaiblit la hiérarchie (recommandation Accedo) ;
- Badges contextuels sur cartes (Nouveau · Gratuit ▶ · Bientôt indisponible) ;
- Hover preview différé ~500 ms, muet, opt-out (D1).

### Erreurs fréquentes
- Mêmes titres répétés dans plusieurs rails ; rails « boîte noire » sans libellé explicable ; grilles sans état d'URL (retour arrière cassé) ; LCP dégradé par un hero lazy-loadé (D1 §6.2).

### Opportunités
- La page **/gratuit** est notre différenciateur : la traiter comme une vitrine (qualité de curation visible, tri par « ajouté récemment »), pas comme un simple filtre.

## 2. Fiches contenus (Film, Série, Saison, Personne, Vidéo UGC)

### Composants indispensables
- Hiérarchie validée D1 : hero backdrop → titre/année/durée/genres/classification → note → **CTA** (▶ Regarder si visionnable · Bande-annonce · + Ma liste · Où regarder) → synopsis → casting → médias → similaires ;
- JSON-LD `Movie`/`TVSeries`/`Person` + fil d'Ariane ;
- Module « Où regarder » neutre avec logos et types d'offre (D5).

### Éléments premium
- **Réponse above-the-fold** pour l'intention SEO (« où regarder X » : le module visible sans scroll) ;
- Sur les séries : reprise contextuelle (« Reprendre S2E4 ») à la place du CTA générique quand une progression existe ;
- Fiches UGC (D11) : gabarit allégé mais même exigence visuelle + bloc source/licence obligatoire.

### Erreurs fréquentes
- CTA noyés sous le synopsis ; bande-annonce en autoplay sonore ; fiches « thin content » sans valeur unique (pénalisées — D1 §6.1) ; oubli de l'état « aucune disponibilité streaming » (prévoir un fallback clair).

### Opportunités
- Le badge « Gratuit ▶ » sur les fiches de titres visionnables est un levier de conversion visite → session lecteur (North Star).

## 3. Page lecteur (Watch) — benchmarks 2025-2026

### Composants indispensables
- Lecteur plein cadre au-dessus de la ligne de flottaison, **fiche allégée dessous** (titre, synopsis court, casting réduit, similaires) — légitime la page en SEO et sert la reprise de session ;
- Contrôles complets validés D7 (dont raccourcis clavier et personnalisation sous-titres) ;
- Pré-roll conditionnel (D6) avec **compte à rebours honnête** (« Votre vidéo dans 5 s ») ;
- Écran de fin : recos post-visionnage annulables, épisode suivant avec compte à rebours (séries).

### Benchmarks de performance à viser (état de l'art mesuré)
| Métrique | Cible |
|---|---|
| Time-to-first-frame | **< 2 s** |
| Rebuffer ratio | **< 1 %** |
| Échec de démarrage vidéo | < 1,5 % |
| INP sur la page Watch | < 200 ms |

### Erreurs fréquentes
- Surcharge d'UI au-dessus du lecteur (la vidéo doit dominer) ; contrôles qui ne disparaissent pas en lecture ; absence de mémorisation du volume/qualité/sous-titres entre sessions ; pré-roll re-servi à chaque reprise du même contenu (frequency capping — D6).

### Opportunités
- **Accessibilité légale** : l'European Accessibility Act est en vigueur depuis juin 2025 — WCAG 2.2 AA sur le lecteur (sous-titres personnalisables, navigation clavier, lecteur d'écran) n'est plus seulement un différenciateur, c'est une obligation pour les services numériques grand public en UE. Nous sommes déjà alignés (D1/D3) — en faire un argument.

## 4. Pages commerciales & support (Contact, FAQ, À propos, légales)

### Composants indispensables
- **FAQ** : accordéons dès ~10 questions, questions issues des vraies demandes (tickets, recherches internes), regroupées par thème (Compte · Visionnage · Publicité · Données personnelles · Upload) ;
- **Contact** : formulaire court (3-4 champs), délai de réponse annoncé, anti-spam invisible (honeypot + rate limit, pas de CAPTCHA agressif) ;
- **À propos** : attributions TMDB/JustWatch (obligation D5), licences du catalogue gratuit, mission ;
- Pages légales lisibles (sommaire ancré, dernière mise à jour visible).

### Point SEO important (recherche 2026)
Google a **déprécié les rich results FAQ (mai 2026)** : plus d'accordéons dans les SERP. Le balisage `FAQPage` reste utile — les crawlers et **les moteurs IA** s'en servent pour extraire des réponses. Décision : conserver le JSON-LD `FAQPage` mais ne plus en attendre de gain visuel SERP ; rédiger les réponses en « format extractible » (réponse directe en 1re phrase).

### Erreurs fréquentes
- FAQ-poubelle (questions inventées pour le SEO) ; formulaire de contact sans confirmation ni suivi ; pages légales en PDF ou en anglais juridique inaccessible.

### Opportunités
- La FAQ **déleste le support** (coût d'un ticket ≫ coût du self-service) — y intégrer les questions publicité/RGPD/upload dès le lancement puisque nos exigences D6/D11 en généreront.

## 5. Transverse : états vides, onboarding, états d'erreur

- **Chaque page privée a un état vide conçu** (règle) : Favoris vides → « Parcourez les tendances et ajoutez votre premier titre » (action unique) ; Historique vide → lien vers /gratuit ; Studio vide `[UGC ON]` → « Téléversez votre première vidéo » + rappel des règles de droits ;
- Un état vide pointe vers **une seule action claire**, pas un menu de possibilités (best practice mesurée : les parcours guidés à action unique augmentent significativement l'activation) ;
- **Onboarding** : le taste picker (D1) est skippable, 3 écrans max, la valeur d'abord (« pour des recommandations qui vous ressemblent ») ;
- **Erreurs** : 404 = recherche + tendances (jamais un cul-de-sac, D1) ; erreur lecteur = message clair + retry + retour fiche ; panne API tierce = contenu servi depuis le cache stale (D1) avec bandeau discret si données datées.

## 6. Ordre de traitement proposé pour la 2.1 (une page à la fois)

1. **Accueil** (la vitrine — mobilise hero, rails, pub display, états connecté/anonyme)
2. **Fiche Film** (le gabarit SEO cœur) → 3. Fiche Série + Saison → 4. **Watch** (+ pré-roll) → 5. Recherche + Résultats → 6. Films/Séries/Genres (grilles facettées) → 7. Découverte/Tendances/Nouveautés → 8. Gratuit ▶ → 9. Ma liste (Favoris/Historique/En cours) → 10. Profil + Paramètres → 11. Studio UGC → 12. Connexion/Inscription/Onboarding → 13. Contact/FAQ/légales → 14. Continuer la lecture (rail + page) — puis 2.2 (Video Experience Specification consolidée).
