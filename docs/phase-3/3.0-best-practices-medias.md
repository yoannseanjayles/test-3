# Phase 3.0 — Best Practices Summary : Médias

> **Livrable** : Synthèse des meilleures pratiques médias (affiches, bannières, hero, miniatures, placeholders, illustrations, médias publicitaires, Open Graph)
> **Statut** : ⏳ En attente de validation HITL
> **Cadrage clé** : les médias de **contenu** (affiches, backdrops, photos) viennent de TMDB (D5) — on ne les génère jamais. La génération IA concerne les médias d'**interface**, de **marque**, de **marketing** et de **publicité interne**.

---

## 1. Médias de contenu (TMDB) — règles d'usage

- **Affiches 2:3** : tailles natives TMDB alignées sur nos breakpoints (`w185` cartes mobiles · `w342` cartes desktop · `w500` fiches) — jamais de redimensionnement arbitraire (netteté + cache CDN, D1 §6.2) ;
- **Backdrops 16:9** : `w780` cartes larges · `w1280` heros ; dégradé de lisibilité **dans le CSS**, pas dans l'image (réutilisabilité) ;
- **Portraits** (`w185`) pour les PersonCard ; logos providers fournis par l'API (D5) ;
- Pipeline : proxy + AVIF/WebP + srcset (D1) ; `alt` selon les règles D1 §6.3.

## 2. Placeholders & états de chargement (le « premium invisible »)

- **LQIP/blurhash** par image : miniature floutée encodée (~30 octets) affichée instantanément — jamais de gris générique ;
- **Couleur dominante extraite** de l'affiche (à l'ingestion, stockée en base) : fond des cartes pendant le chargement + hero fallback des titres sans backdrop (D15) ;
- Placeholders **sans image du tout** (titre sans affiche TMDB, fréquent sur les titres anciens) : tuile générée = dégradé de la couleur du genre + monogramme du titre — à générer en Phase 3.2 (série de fonds par genre) ;
- Ratio toujours réservé (`aspect-ratio`) : le placeholder a exactement la taille de l'image finale (CLS 0).

## 3. Médias d'interface & de marque (à générer — périmètre IA)

| Famille | Usage | Contraintes |
|---|---|---|
| **Illustrations d'états vides** | Favoris/Historique/Studio vides, 404, erreur lecteur (D13/D19) | Style unique cohérent, dark-mode-first, léger (SVG ou PNG optimisé), ton chaleureux jamais infantilisant |
| **Cartes de genres** | Hub /genres, /decouvrir (D19) | 16:9, évocation du genre **sans référence à une œuvre réelle** (droits !), texte ajouté en CSS |
| **Fonds « chips d'intention »** | /decouvrir (H34) | Ambiances abstraites (frissons, famille, rire) |
| **Visuels onboarding** | 3 écrans /bienvenue | Cohérents avec les états vides |
| **Hero génériques** | /gratuit (fallback), pages support | Ambiance cinéma (salle, projecteur, pellicule) sans personnage identifiable |
| **Logo & favicon** | Marque (naming H4, Phase 4) | Déclinaisons : plein, monogramme, monochrome |

## 4. Médias publicitaires (D6)

- **House ads** (autopromo) pour le lancement : créas aux formats exacts des emplacements (display responsive, carte native = gabarit TitleCard/WideCard) — elles valident le pipeline pub sans annonceur et évitent les espaces vides pub ON ;
- Carte native sponsorisée : **mêmes dimensions que les cartes contenu**, étiquette « Sponsorisé » réservée en zone non masquable (Advertising Design System, Phase 4) ;
- Pré-roll de démonstration : 1-2 vidéos house 10-15 s (validation du pipeline VAST en 6.2) ;
- Jamais de créa imitant l'UI du site (dark patterns interdits — D6).

## 5. Open Graph & partage (état de l'art 2026)

- **1200×630** (1,91:1) universel ; zone de sécurité centrale **1080×600** (marges 60-80 px) ; < 1 Mo (WhatsApp compresse au-delà de ~300 Ko — viser léger) ;
- **Optimiser pour le petit écran** : gros corps de texte, contraste fort — pas de finesse typographique (compressée, elle disparaît) ;
- **Génération dynamique par titre** (D15) : template composé (backdrop + affiche + titre + note + logo marque) rendu au build/publish, URL stable, `og:image:width/height/alt` renseignés ;
- Templates OG statiques pour les pages non-titre (accueil, /gratuit, listes) ;
- X/Twitter : `summary_large_image` accepte 1200×630 ; pas de déclinaison Pinterest en v1.

## 6. Génération IA — règles de prompt (pour la 3.2)

- **Structure CRAFT** : Contexte (sujet+décor) · Rendu (style/médium) · Atmosphère (humeur+lumière) · Fidélité (cadrage/composition) · Tool modifiers (ratio, négatifs) ;
- **20-50 mots** : au-delà de ~80 mots la qualité se dégrade ; concision précise > verbosité ;
- **Par modèle** : Midjourney = phrases courtes à haut signal + `--ar` ; **Flux** = langage naturel long et littéral (photoréalisme) ; **Ideogram** = seul fiable pour du **texte lisible dans l'image** ; GPT Image = bon suiveur d'instructions multi-éléments ;
- **Ratio déclaré avant génération** (jamais de recadrage destructif après coup) ; générer en 2× la taille cible ;
- **Cohérence de série** : même bloc de style (palette, grain, éclairage) réutilisé mot pour mot dans tous les prompts d'une famille + génération par lots avec seed/référence quand le modèle le permet ;
- **Espace texte** : prévoir la zone vide (« large negative space on the left for text overlay ») — le texte définitif est posé en CSS/design, pas généré (sauf Ideogram pour les créas marketing) ;
- **Interdits légaux** : aucune ressemblance avec des acteurs réels, personnages, affiches ou logos d'œuvres existantes ; aucune imitation de marque de streamer ; style « inspiré de » générique uniquement.

## 7. Erreurs fréquentes à éviter

Texte généré illisible (hors Ideogram) · styles incohérents entre écrans (pas de bloc de style commun) · images trop lourdes non converties AVIF/WebP · OG « jolie mais illisible » en vignette · visuels de genre utilisant des œuvres reconnaissables · placeholders gris qui cassent l'ambiance dark premium · illustrations d'états vides culpabilisantes.

## Hypothèses signalées

- **H52** : la génération est faite hors plateforme (ChatGPT Images/Midjourney/Flux/Ideogram selon la famille) puis versionnée dans le repo — pas d'API de génération intégrée au produit en v1.
- **H53** : le style visuel définitif des illustrations dépend de l'identité (Phase 4/H4) ; la 3.2 livrera des prompts **paramétrés** (`{PALETTE}`, `{STYLE_BLOCK}`) réutilisables après le choix d'identité.
