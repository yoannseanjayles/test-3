# Phase 4.0 — Best Practices Summary : Design System

> **Livrable** : Synthèse des meilleures pratiques (design systems, interfaces streaming/TV, web premium, motion, publicité non intrusive)
> **Statut** : ⏳ En attente de validation HITL
> **Complète** : D1 §6.4 (design premium), D13 (pages), D20 (lecteur) — sans les répéter.

---

## 1. Design tokens (le socle — état de l'art 2026)

- **Spécification W3C Design Tokens stabilisée** : format standard supporté par Figma Variables, Style Dictionary, etc. — nos tokens seront définis dans ce format et compilés en variables CSS ;
- **3 couches de tokens** : primitifs (`violet-500`) → **sémantiques** (`surface-raised`, `text-primary`, `accent-brand`) → composants (`card-bg`) ; seuls les sémantiques sont utilisés dans les composants — c'est la seule approche qui permet le thème clair futur (Could-have D4) sans refonte ;
- **Contraste vérifié à la création du token** : `text-primary` sur `surface-base` ≥ 4,5:1 (AA) dans chaque thème avant publication — le contraste est une propriété du système, pas un correctif ;
- Une convention de nommage unique design ↔ code + tokens versionnés dans le repo.

## 2. Thème sombre premium (par défaut — D1)

- **Jamais de noir pur** : fond de départ `~#141118` (notre charbon, déjà utilisé en 3.2), puis **échelle de surfaces** : 3-4 paliers de +5-8 % de luminance chacun (`surface-base` → `raised` → `overlay`) — la profondeur se lit par la lumière, pas par les ombres ;
- Texte off-white (jamais #FFF pur — halo), hiérarchie par opacité (87 % / 60 % / 38 %) ;
- **Un accent de marque unique** (D14 : CTA primaire) + sémantiques fonctionnels (succès/erreur/alerte) désaturés pour le fond sombre ;
- Lueurs douces et paliers de luminance plutôt qu'élévations à ombres portées (illisibles sur sombre) ;
- Les affiches/backdrops sont la matière colorée principale : l'UI reste neutre et s'efface (leçon des leaders — D1 §6.4).

## 3. Composants & interfaces streaming (rappels critiques)

- **Stabilité d'abord** : hauteurs réservées partout (CLS 0 — D1), skeletons systématiques, `aria-live` pour les mises à jour dynamiques, focus jamais perdu quand le contenu se rafraîchit ;
- **États de récupération conçus** : panne = message amical + retry auto avec backoff (déjà spécifié D17 §9) — le design system fournit le composant ;
- Bibliothèque à documenter (base : composants identifiés en 2.1) : boutons (3 niveaux), TitleCard/WideCard/ProgressCard/PersonCard/EpisodeRow, Rail, facettes/chips, champs, modales, toasts, badges, skeletons, lecteur (habillage Vidstack), tableaux admin ;
- Chaque composant est spécifié avec **tous ses états** : repos, hover, focus visible, actif, désactivé, chargement, erreur, vide.

## 4. Motion (langage d'interaction, pas décoration)

- **Courbes d'easing custom** (jamais de linéaire) : standard `cubic-bezier(0.2, 0, 0, 1)` pour les entrées, sorties plus rapides que les entrées ;
- Durées : micro-interactions 150-250 ms (D1) · transitions de vue 250-350 ms · jamais > 500 ms ;
- Le motion **communique** : un élément qui glisse hors écran dit « supprimé », un skeleton qui pulse dit « ça arrive » — chaque animation a une fonction, sinon elle saute ;
- **`prefers-reduced-motion` structurel** (~35 % des adultes de 40+ ont une sensibilité vestibulaire) : chaque token de motion a sa variante réduite (fondu simple) — définie dans le système, pas au cas par cas ;
- Rythme homogène : mêmes courbes/durées sur mobile, desktop et 10-feet (D20).

## 5. Advertising Design System (D6 — exigence du brief)

- **Principe : la pub est un citoyen du design system, pas un iframe sauvage** — les formats display sont encadrés dans des conteneurs aux dimensions **réservées** (zéro CLS publicitaire, le pire anti-pattern du web ad-supported) ;
- **Étiquetage** : token dédié `ad-label` (« Publicité » / « Sponsorisé »), position fixe non masquable, contraste AA — jamais de mimétisme avec le contenu éditorial au-delà du gabarit de carte (D10 §3) ;
- Cartes sponsorisées : même grille/gabarit que TitleCard/WideCard (intégration visuelle) mais **bordure/fond `surface-sponsored`** subtilement distincts + étiquette ;
- Pré-roll : habillage du lecteur défini en D20 §7 (compte à rebours, skip) — le design system fournit les composants `AdCountdown`, `SkipButton`, `AdLabel` ;
- Densité plafonnée par gabarits (déjà cartographiée D10 §3) ; états « pub OFF » = composants absents (D14) ;
- House ads (3.1/D) suivent les mêmes règles que les pubs externes — aucun privilège.
