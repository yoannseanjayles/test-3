## 1. Identité

### 1.1 Nom — décision H4 (tranchée D25)
**Nom définitif : Ciné+ (graphie alternative : CinePlus)** — décision du commanditaire (D25). Le « + » s'intègre naturellement au logomark (play/aperture + signe plus) et évoque le « cinéma augmenté » (découverte + visionnage + suivi).
⚠️ **Risque juridique signalé (R6)** : « Ciné+ » est une marque existante du groupe Canal+ (bouquet de chaînes cinéma françaises). Une vérification INPI/EUIPO et un avis juridique sont **indispensables avant toute mise en production publique** ; la graphie « CinePlus » n'élimine pas le risque de confusion dans le même secteur. En attendant, le nom est utilisé dans le code et les maquettes (remplaçable par token de marque unique).

### 1.2 Logo
Direction validée par l'exploration 3.2 : **triangle play inscrit dans une ouverture d'objectif** (aperture), géométrique, monochrome-compatible. Déclinaisons : logomark seul (favicon/app), logotype horizontal (header), version monochrome (footer, partenaires). Vectorisation manuelle à partir de la génération retenue — à produire avec le lot médias reporté (D24).

## 2. Fondations

- **Espacement** : échelle 4 px (`space-1`=4 … `space-12`=96) ; gouttières grilles 16 (mobile) / 24 (desktop) ;
- **Rayons** : `radius-s` 6 (chips, badges) · `radius-m` 10 (cartes, champs) · `radius-l` 16 (modales, hero) · `radius-full` (avatars, boutons pill) ;
- **Élévation sombre** : par paliers de surface (D24) + lueur `glow-accent` (CTA hover) ; ombres portées réservées aux overlays (`shadow-overlay`) ;
- **Z-index** : `base 0 · rail-controls 10 · header 100 · dropdown 200 · toast 300 · modal 400 · player-fullscreen 500 · cmp 600` ;
- **Breakpoints** : `sm 360 · md 768 · lg 1024 · xl 1440 · tv 1920+` (mode 10-feet D20 §4) ;
- **Motion** : `duration-fast 150ms · base 250ms · slow 350ms` ; easings `ease-enter cubic-bezier(0.2,0,0,1)` · `ease-exit cubic-bezier(0.4,0,1,1)` ; chaque usage a sa variante `prefers-reduced-motion` (fondu 100 ms) — définie au niveau token ;
- **Focus** : ring ambre 2 px + offset 2 px, visible sur fond sombre ET sur images (double liseré) ; cibles ≥ 24 px desktop / 44 px tactile.

## 3. Bibliothèque de composants (spécification des états)

Tous les composants déclinent : **repos · hover · focus-visible · actif · désactivé · chargement (skeleton) · erreur · vide** (quand applicable). Sélection structurante :

| Composant | Points clés |
|---|---|
| `Button` (primary/secondary/ghost + icon) | primary = ambre/on-accent, pill ; loading = spinner inline sans changement de largeur |
| `TitleCard` (2:3) / `WideCard` (16:9) / `ProgressCard` / `PersonCard` / `EpisodeRow` | fond `surface-raised`, blurhash → image, hover scale 1,04 + glow (250 ms, désactivé reduced-motion), badges en coin, 1 lien englobant + actions au focus (D1) |
| `Rail` | scroll horizontal, flèches ≥ 24 px, `role=region`+`aria-label`, scroll-snap, masque de fondu latéral |
| `FacetBar` + `Chip` | chips actives ambre outline, compteurs `text-secondary`, tiroir mobile plein écran |
| `SearchOverlay` | fond `surface-overlay` + backdrop-blur, navigation clavier complète (D19) |
| `Modal` / `TrailerModal` | focus-trap, Échap, retour focus, `radius-l` |
| `Toast` | 5 s, action « Annuler », `aria-live=polite`, pile max 3 |
| `Tabs`, `Field`, `Select`, `Toggle`, `Skeleton`, `Badge`, `Avatar`, `Breadcrumb`, `EmptyState` (illustration B3 + 1 action — D13) | conformes fondations §2 |
| Lecteur (habillage Vidstack) | thème tokens (accent = progression), contrôles D17/D20, timecodes JetBrains Mono |
| Admin : `DataTable`, `StatCard`, `QuotaGauge`, `ModerationCard` | densité renforcée, mono tabulaire pour les chiffres |

## 4. Advertising Design System (D6/D24 §5)

| Composant | Spécification |
|---|---|
| `AdSlot` (display) | conteneur à **dimensions réservées** par breakpoint (zéro CLS pub) ; fond `surface-base` + liseré `border-subtle` ; étiquette obligatoire ; pub OFF/non servie → composant **absent du DOM** (jamais de cadre vide — D14) |
| `AdLabel` | « Publicité » / « Sponsorisé », `caption` 12 px, `ad-label-bg` (violet 15 %) + `ad-label-text` (AA), coin supérieur, non masquable |
| `SponsoredCard` | gabarit TitleCard/WideCard sur `surface-sponsored` (teinte violette 6 %) + AdLabel — intégrée mais toujours identifiable |
| `AdCountdown` / `SkipButton` (pré-roll) | « Votre vidéo dans Xs » text-secondary ; skip = pill ghost, activable à 5 s avec compte à rebours dans le bouton, cible 44 px (D20 §7) |
| Interdits (rappel) | mimétisme UI, autoplay sonore, overlay sur contenu, pub dans espaces interdits (D10 §3) |

## 5. Guidelines transverses

- **Imagerie TMDB** : dégradés de lisibilité standardisés (`scrim-hero` : linéaire 0→85 % charbon ; `scrim-card`), jamais de texte sur zone non assombrie ; ratios réservés partout ;
- **Iconographie** : [Lucide](https://lucide.dev) (open source, 2 px stroke, cohérente avec Space Grotesk) + pictos badges custom (B11) ;
- **Ton rédactionnel FR** : vouvoiement, phrases courtes, jamais culpabilisant (états vides = invitation), techniquement honnête (erreurs, pubs, données — D17/D6/RGPD) ; libellés d'action à l'infinitif court (« Regarder », « Reprendre ») ;
- **Accessibilité par composant** : chaque spec §3 embarque ses exigences ARIA/clavier (issues de D1 §6.3, D20 §8) — l'a11y est dans la définition of done de chaque composant, pas une passe finale.
