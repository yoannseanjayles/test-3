# Phase 6.2 — Advertising Technical Specification

> **Étape** : 6.2 · **Statut** : ⏳ En attente de validation HITL (P38) — clôture de la Phase 6
> **S'appuie sur** : D3 (pub plafonnée VAST/VMAP + TCF), D6 (kill switch + pré-roll exigence ferme), D17 (pré-roll seule pub de Watch, capping), D25 §4 (Advertising DS : conteneurs réservés, AdLabel), 2.2 (contrat pub in-player), 6.1 Lot 4 (pilotage back-office)
> **Principe cardinal (D3 n°15)** : aucun placement ne dégrade l'expérience — la plateforme reste pleinement fonctionnelle publicité OFF (état par défaut).

## 1. Ce qui existe déjà (acquis 5.1/6.1)

L'infrastructure d'accueil de la publicité est **déjà en production** — cette spécification décrit ce qui s'y branche :

- **AdSlots à hauteur réservée** (`display.home/browse/fiche`, `native.rail`) : absents du DOM si pub OFF (zéro CLS, D14) ;
- **Pré-roll in-player** : séquence D17 implémentée (jamais à la reprise, capping 1/titre/session en `sessionStorage`, skippable après 5 s) avec placeholder maison ;
- **Pilotage runtime** : 7 interrupteurs en base, effet ≤ 30 s, kill switch env prioritaire (D6), écran back-office dédié ;
- **Test bloquant** : « Publicité » absente du DOM quand OFF (suite E2E).

## 2. Chaîne publicitaire cible

```
AdSlot / adSlot player
  → Ad Decision (lib/ads/decision.ts — serveur) :
      kill switch D6 → consentement TCF → capping/pression → sélection
  → v1 : house ads (auto-promo, assets B7) servies localement
  → v2 : VAST 4.x tag (ad server / SSP) résolu côté serveur, média proxifié
  → rendu (display : image/HTML sandboxé · vidéo : média VAST dans le player)
  → mesure : impression / quartiles / complétion / skip / viewabilité
```

**Décision proposée : v1 en régie directe (house ads + campagnes vendues de gré à gré, servies par nous) ; v2 en programmatique (VAST tags tiers).** Justification : un SSP/ad exchange exige une audience préalable ; la régie directe monétise dès le premier annonceur local sans dépendance, avec la même tuyauterie de mesure.

## 3. Formats & règles de pression (plafonds D3/D17)

| Emplacement | Format | Règles fermes |
|---|---|---|
| `video.preroll` | Vidéo ≤ 30 s, skippable après 5 s (D6) | **Seule pub de Watch** (D17) · 1/titre/session · jamais à la reprise · jamais sur épisode suivant enchaîné < 30 min · pas de mid-roll/post-roll en v1 |
| `display.home` | 970×250 / 728×90 (desktop), 320×100 (mobile) | 1 par page max, sous le pli du hero, hauteur réservée |
| `display.browse` / `display.fiche` | 728×90 / 300×250 | 1 par page max, jamais entre le CTA et le contenu |
| `native.rail` | Carte sponsorisée au format TitleCard | Max 1 carte/rail, ≤ 1 rail sponsorisé/page, `AdLabel` « Sponsorisé » obligatoire (D25 §4) |

**Pression globale** : ≤ 2 emplacements display+natif par page (D14) ; fréquence pré-roll pilotable en base (`ads.preroll.cap`, Phase 7 quand `app_config` passera aux valeurs numériques — H96).

## 4. Consentement — TCF v2.2

- **CMP** : à l'activation de la pub, bandeau de consentement (accepter / refuser / personnaliser, symétriques — CNIL) ; le bouton « Gérer mes préférences » de `/cookies` le rouvre (déjà en place, désactivé) ;
- **Règle stricte** : pas de consentement publicité ⇒ **aucun cookie/traceur pub** ; en v1 régie directe, une pub *non ciblée et non tracée* (contextuelle) reste servable sans consentement (base légale : intérêt légitime, sans données personnelles — à confirmer en revue juridique H46) ; en v2 programmatique, la TC String est transmise dans les macros VAST ;
- CMP candidate free tier : implémentation maison v1 (choix binaire + granularité analytics/pub, stockage 1re partie 6 mois), bascule vers une CMP IAB enregistrée en v2 si programmatique (obligation TCF).

## 5. Mesure & revenus

- Événements (schéma `ad_events`, ajout Phase 7) : `ad.request / ad.fill / ad.impression / ad.quartile(25|50|75) / ad.complete / ad.skip / ad.click / ad.error` + `placement`, `campaignId`, session anonyme ;
- **Viewabilité display** : IntersectionObserver ≥ 50 % pendant ≥ 1 s (MRC) ; vidéo : lecture effective ≥ 2 s ;
- Tableau **Revenus** du back-office (Phase 7, H95) : impressions/complétion/eCPM par campagne — alimente le Business Model Canvas (Phase 8) ;
- Anti-fraude v1 : mesure serveur (pas de pixel tiers), déduplication par session, exclusion des admins.

## 6. Brand safety & qualité

Pas de pub sur : pages d'erreur, pages légales/contact, Studio, back-office, contenus en modération. Créas : poids ≤ 150 Ko display (image optimisée servie par `next/image`), vidéo pré-roll ré-encodée par notre pipeline (jamais de VAST média direct en v1), aucun script tiers dans les créas (iframe sandboxée si HTML). Catégories exclues et plafond de répétition par campagne configurés en base.

## 7. Plan d'implémentation par paliers

| Palier | Contenu | Dépendances |
|---|---|---|
| **P-1 (Phase 7)** | `ad_events` + tableau Revenus + `lib/ads/decision.ts` (décision servie : house ads) + CMP maison + créas B7 (house ads) | Base active ; médias B7 (reportés D24) |
| **P-2** | Campagnes directes : table `campaigns` (dates, plafonds, créas, ciblage contextuel grossier), rotation pondérée, capping par campagne | P-1 + premiers annonceurs (kit média Phase 8) |
| **P-3 (v2)** | VAST 4.x : résolution serveur du tag, tracking events VAST, TC String, CMP IAB enregistrée, wrapper ≤ 2 niveaux | Audience mesurable + revue juridique H46 + licence TMDB commerciale (R1) |

## Hypothèses signalées

- **H97** : v1 = régie directe sans SSP (voir §2) — la trajectoire programmatique est préservée par le contrat `decision.ts` ;
- **H98** : la pub contextuelle sans traceur est servable sans consentement TCF (à confirmer en revue juridique H46 avant activation) ;
- **H99** : R1 (licence commerciale TMDB) doit être levé **avant** toute activation effective de la publicité — la monétisation déclenche l'obligation ;
- **H100** : les assets house ads (famille B7) restent à générer (reportés en D24) — prérequis du palier P-1.
